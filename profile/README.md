## 一、先别急着换IP——可能只是虚惊一场

很多人IP连不上，第一反应就是"IP被封了"。但实际上，导致VPS连不上的原因挺多的，不一定是IP层面的问题。

在做搬瓦工IP测试之前，先排查一下这几个更常见的情况：

**1. VPS本身宕机了**
搬瓦工偶尔会有机器宕机的情况，尤其是某个机房集中出问题的时候。可以登录搬瓦工KiwiVM控制面板，看看VPS状态是不是Running。如果显示stopped，直接点启动就好了，跟IP没关系。

**2. SSH端口被封（不是IP整体被封）**
有时候是具体的SSH端口（默认22或者自定义端口）被防火墙识别拦截，但IP本身其实没问题。这种情况下换个端口可能就解决了。

**3. 本地网络问题**
自己的宽带偶尔抽风、运营商临时路由波动，也会导致连不上。试着用手机热点换个网络测一下，如果手机流量能连、家里宽带连不上，那问题在本地，不在IP。

排查完以上几种情况，还是连不上，再进行正式的搬瓦工IP测试。

## 二、搬瓦工IP测试：三种方法

### 方法一：KiwiVM官方自带检测（最权威）

这是搬瓦工官方提供的IP黑名单检测工具，结果最可信。

**操作步骤：**

1. 登录搬瓦工账户，进入 Services → My Services，找到对应的VPS实例
2. 点击进入 KiwiVM Control Panel
3. 在KiwiVM控制面板的URL中，将末尾的 `main.php` 替换为 `main-exec.php?mode=blacklistcheck`
4. 进入检测页面后，点击 **Test Main IP** 按钮

**结果解读：**
- **IP NOT BLOCKED**：IP没问题，连不上另有原因
- **IP BLOCKED（红色加粗）**：IP确实被封了，需要处理

如果检测结果里出现了 `pinger offline`，那是测试节点自身的问题，跟你的VPS无关，可以忽略。

### 方法二：第三方端口检测工具

如果你暂时进不了KiwiVM，可以用第三方工具先做个初步判断。

找一个支持检测国内/国外连通性的工具，输入你的VPS IP和SSH端口（比如22或者自定义端口），看检测结果：

- **国外可用 + 国内不可用**：基本可以判定IP在国内被墙了
- **国内外都不可用**：端口没开放，检查VPS防火墙设置
- **国内外都可用**：IP和端口都正常，问题在别处

### 方法三：多地ping测试

使用ping.pe等工具，输入IP地址，查看各地节点的ping结果：

- 如果海外节点都能ping通，只有中国大陆的节点超时或丢包严重，那就是IP在大陆被封了
- 如果全球都ping不通，VPS可能已经宕机或者网络出了别的问题

## 三、确认IP被封之后，怎么处理

搬瓦工IP测试确认IP已被封，目前有以下几个处理思路。

### 方案一：等待自然解封

这是成本最低的方式，但也是最被动的选择。

一般来说，如果你什么操作都不做，静静等上2到3个月，IP可能会自然解封。之所以说"可能"，是因为不同时期、不同情况的解封时间差异很大，有人等了一周就好了，也有人等了半年依然没动静。
